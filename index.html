03 — Works

💼 취업 지원 웹 서비스
PortfoLink — 3-Tier 인프라 구축·운영

2026.06 — 2026.07
팀 프로젝트 · Raspberry Pi 2B · Ubuntu 22.04

Overview

채용 공고 열람, 지원 현황 관리, 포트폴리오 작성·공개까지 제공하는 취업 지원 웹 서비스입니다. 라즈베리파이 한 대 위에 WEB(Nginx) — WAS(FastAPI) — DB(MariaDB) 3-tier 구조를 직접 구축하고 도메인 연결부터 배포·운영까지 전 과정을 담당했습니다.

차별화 기능으로, 회원이 포트폴리오를 HTML 파일로 업로드·편집하고 개인 도메인(CNAME)을 연결하면 자신의 도메인으로 포트폴리오가 서빙됩니다. Nginx가 Host 헤더를 백엔드로 넘기고, 백엔드가 회원별 포트폴리오 HTML을 찾아 응답하는 구조입니다.

브라우저 ──> Nginx (:80, two.greatsounds.me) ├─ /api/* ──> FastAPI (127.0.0.1:8000) ──> MariaDB ├─ /uploads/portfolio/* ──> 정적 서빙 (no-cache) ├─ "/" (개인 도메인 접속) ──> /api/portfolios/by-host └─ 그 외 html/css/js ──> WEB_ROOT 직접 서빙 ✓ 백엔드 health check ... OK ✓ nginx -t & restart ... OK

What I Built

01인프라 구축 & 배포 자동화

Nginx 설치·서버 블록 생성·정적 파일 배포·리버스 프록시 설정을 멱등(idempotent) 쉘 스크립트로 자동화. start_all / restart_all / stop_all / clean_all 등 서비스 전체 생명주기를 스크립트로 관리

02백엔드 API & 인증

FastAPI + MariaDB 기반 REST API 구현. httpOnly 쿠키에 JWT를 담는 인증 설계(1GB RAM 환경을 고려해 Redis 세션 대신 선택), 로테이션 파일 로깅과 로그 관제 스크립트로 4xx/5xx 실시간 모니터링

03크롤러 & 알림 자동화

잡코리아 공고 크롤러를 crontab(매일) + 로그인 트리거(BackgroundTasks) 두 경로로 운영. crawl_log 테이블을 락으로 활용한 중복 실행 방지·쿨다운 설계, 신규 공고를 디스코드 봇으로 자동 알림

What I Learned

저사양 서버 한 대에서 WEB·WAS·DB 전 계층을 직접 구축·운영하며, 리버스 프록시·DNS·로그 관제·멱등 스크립트 설계 등 인프라 운영의 전체 흐름을 손으로 익혔습니다. 제약(1GB RAM, HTTP 전용)을 인지하고 그 안에서 근거 있는 기술 선택을 하는 경험을 했습니다.

🛡️ 시스템 보안점검
프로그램 제작

2026.05 — 2026.06
개인 프로젝트 · Ubuntu 24.04 LTS

Overview

KISA(한국인터넷진흥원)의 주요정보통신기반시설 기술적 취약점 분석·평가 가이드를 기반으로, 리눅스 시스템의 보안 취약점 72개 항목을 자동으로 점검하는 프로그램을 제작했습니다.

코드 작성 자체보다 점검 대상 파일과 시스템의 동작을 이해하는 것을 목표로, 점검 결과에 조치 파일·조치 방법까지 함께 출력해 실제 운영에서 바로 활용할 수 있도록 설계했습니다.

======================================== Linux 보안 점검 스크립트 ======================================== [U-01] root 계정 원격 접속 제한 ... [취약] 조치 파일: /etc/ssh/sshd_config 조치 방법: PermitRootLogin no 로 변경 [U-02] 패스워드 최소 길이 설정 ... [양호] [U-03] 계정 잠금 임계값 설정 ... [양호] ======================================== 점검 완료 | 양호: 2 | 취약: 1 | 총 72개 ========================================

Development Phases

v1개별 파일 점검

항목별 쉘 스크립트로 텍스트 비교, stat 기반 권한 점검, 파일 존재 확인 로직 구현

v2통합 함수화

점검 로직을 TXT_COMP · PERM_CHK · FILE_EXT 타입별 공통 함수로 통합하고, 점검 항목을 cfg 설정 파일로 분리해 유지보수성 개선

v3파이썬 원격 점검

paramiko 기반 SSH 연결로 로컬에서 cfg와 스크립트를 조립해 원격 서버에 전송·실행하고, 날짜별 리포트(report_YYYYMMDD)를 자동 생성. 다수 서버 일괄 점검 구조로 확장

What I Learned

반복되는 점검 코드를 함수와 설정으로 분리하는 구조화 과정, 그리고 "실행 위치가 원격 서버"라는 제약을 스크립트 조립·전송 방식으로 해결하며 자동화 도구를 설계하는 사고방식을 익혔습니다.
